# AI Agent를 만들려다 생긴 일

## 시트콤 대사로 영어 공부하는 카카오톡 봇

---

# AI Agent란?

> **목표를 주면 스스로 판단하며 여러 단계를 실행하는 프로그램**

```
목표: "오늘 영어 표현 보내줘"
  → 어떤 시트콤을 볼까? (판단)
  → 이 대사는 학습에 별로네, 다시 골라야겠다. (판단)
  → 전송에 실패했네, 재시도해야겠다. (판단)
  → 완료
```

핵심은 **루프 + 자율 판단**

---

# AI Agent의 3가지 구성요소

**뇌 (LLM)** + **도구 (Tools)** + **루프 (Loop)**

| 구성 | 역할               | 예시                      |
| ---- | ------------------ | ------------------------- |
| 뇌   | 다음 행동 결정     | Claude, GPT               |
| 도구 | 실제 행동 수행     | API 호출, 검색, 코드 실행 |
| 루프 | 목표 달성까지 반복 | 결과 평가 → 재시도        |

---

# 처음 목표

카카오톡으로 매일 영어 표현을 자동으로 받아보고 싶다

- 실생활 영어 회화 표현 학습
- 시트콤 대사 기반 (Friends, The Office 등)
- 하루 3번 자동 전송
- AI가 번역 + 설명 생성

---

# 결과물: AI 자동화 파이프라인

에이전트가 아닌, **AI를 활용한 자동화 스크립트**

```
GitHub Actions (매일 8시, 1시, 7시, 11시)
        │
        ▼
OpenSubtitles API → 시트콤 자막에서 대사 수집
        │
        ▼
Claude API → 번역 + 핵심 표현 + 실생활 예문 생성
        │
        ▼
카카오 메시지 API → 내 카카오톡으로 전송
```

# 에이전트 vs 우리가 만든 것

|           | AI Agent        | 우리 결과물 |
| --------- | --------------- | ----------- |
| 흐름 결정 | Claude가 스스로 | 코드에 고정 |
| 재시도    | 자율 판단       | 없음        |
| 복잡도    | 높음            | 낮음        |
| 비용      | 높음            | 낮음        |
| 안정성    | 예측 어려움     | 예측 가능   |

---

# 개발 순서

**Step 1** 카카오 API 연동
→ 나에게 보내기 테스트

**Step 2** OpenSubtitles API 연동
→ 시트콤 자막 다운로드 및 파싱

**Step 3** Claude API 연동
→ 대사 분석, 번역, 예문 생성

**Step 4** 세 모듈 통합
→ 전체 플로우 테스트

**Step 5** GitHub Actions 배포
→ 스케줄 자동 실행

---

# 학습 효과를 높이기 위한 고민

단순히 받아보는 것 → **능동적 학습**으로

```
❌ 기존: 영어 원문 → 한국어 번역 → 설명

✅ 개선: 한국어 힌트만 먼저 → 영어로 생각해보기
         → 정답 + 핵심 표현 + 예문 확인
```

밤 11시 복습 메시지로 반복 효과

# 에이전트로 발전시킨다면?

지금은 흐름이 코드에 고정되어 있지만

- 대사 품질을 Claude가 직접 평가하고 재시도
- 학습자 수준에 맞게 난이도 자동 조절
- 이전에 배운 표현 기억해서 중복 방지
- "오늘 내 수준에 맞는 표현 알아서 찾아줘" 같은 열린 목표 처리

**복잡도와 비용이 올라가는 만큼, 진짜 필요할 때 도입하는 게 맞다**
